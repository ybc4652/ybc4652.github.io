2. 设置开机启动#

# 检查开启启动状态，disabled为开机不自动启动$ systemctl list-unit-files|grep mysqldmysqld.service disabledmysqld@.service disabled
# 设置开启启动$ systemctl enable mysqld.serviceCreated symlink /etc/systemd/system/multi-user.target.wants/mysqld.service → /usr/lib/systemd/system/mysqld.service.
# 检查开机启动状态，enabled为开机不自动启动$ systemctl list-unit-files|grep mysqldmysqld.service enabledmysqld@.service disabled

3. 启动MySql服务#

# 检查是否启动MySql服务，无mysql即为未启动$ ps -ef|grep mysqlroot       26597    1646  0 14:28 pts/0    00:00:00 grep --color=auto mysql
# 启动MySql服务$ systemctl start mysqld.service
# 查看MySql是否启动，出现mysql即为启动成功$ ps -ef|grep mysqlmysql      26707       1 15 14:30 ?        00:00:01 /usr/libexec/mysqld --basedir=/usrroot       26784    1646  0 14:30 pts/0    00:00:00 grep --color=auto mysql

6. 设置远程连接#

此时MySql客户端还无法连接上MySql服务，需要MySql服务开启远程连接。

# 切换数据库mysql> use mysqlReading table information for completion of table and column namesYou can turn off this feature to get a quicker startup with -A
Database changed
# 查看user表mysql> select host,user,plugin from user;+-----------+------------------+-----------------------+| host      | user             | plugin                |+-----------+------------------+-----------------------+| localhost | mysql.infoschema | caching_sha2_password || localhost | mysql.session    | caching_sha2_password || localhost | mysql.sys        | caching_sha2_password || localhost | root             | mysql_native_password |+-----------+------------------+-----------------------+4 rows in set (0.00 sec)
# 允许root用户远程连接mysql> update user set host='%' where user='root';Query OK, 1 row affected (0.00 sec)Rows matched: 1  Changed: 1  Warnings: 0
# 查看user表查看修改是否成功mysql> select host,user from user;+-----------+------------------+-----------------------+| host      | user             | plugin                |+-----------+------------------+-----------------------+| %         | root             | mysql_native_password || localhost | mysql.infoschema | caching_sha2_password || localhost | mysql.session    | caching_sha2_password || localhost | mysql.sys        | caching_sha2_password |+-----------+------------------+-----------------------+4 rows in set (0.00 sec)
# 退出mysqlmysql> exit;Bye
# 重启服务[root@iZ2ze6377ws99tqt8gh5shZ /]# systemctl restart mysqld.service

7. 防火墙添加规则#

若防火墙启用状态需要允许外网访问3306端口

# 查看防火墙状态, Active为inactive (dead)则为未开启$ systemctl status firewalld● firewalld.service - firewalld - dynamic firewall daemon   Loaded: loaded (/usr/lib/systemd/system/firewalld.service; disabled; vendor preset: enabled)   Active: active (running) since Fri 2021-10-15 15:38:00 CST; 2s ago     Docs: man:firewalld(1) Main PID: 27401 (firewalld)    Tasks: 2 (limit: 10882)   Memory: 29.4M   CGroup: /system.slice/firewalld.service           └─27401 /usr/libexec/platform-python -s /usr/sbin/firewalld --nofork --nopid
# 添加3306端口到防火墙规则$ firewall-cmd --permanent --zone=public --add-port=3306/tcpsuccess
# 检查3306是否添加到防火墙规则里$ firewall-cmd --query-port=3306/tcpyes
#重启防火墙服务$ systemctl restart firewalld.service

8. 结语#

至此，CentOS8安装MySql8完成。若客户端依旧连接不上MySql服务，请参考 客户端连接MySQl8失败的解决方案
